# 贡献指南 / Contributing Guide

感谢你对这个项目的关注！/ Thank you for your interest in this project!

## 如何贡献 / How to Contribute

### 提出建议 / Suggestions
如果你有任何建议或想法，欢迎：
- 提交 Issue 讨论你的想法
- 直接联系我进行交流

If you have suggestions or ideas, feel free to:
- Open an Issue to discuss your ideas
- Contact me directly

### 报告问题 / Reporting Issues
如果你发现了任何问题：
1. 检查是否已经有相关的 Issue
2. 如果没有，创建一个新的 Issue
3. 详细描述问题和复现步骤

If you find any issues:
1. Check if there's already a related Issue
2. If not, create a new Issue
3. Provide detailed description and reproduction steps

### 提交改进 / Submit Improvements
如果你想直接改进内容：
1. Fork 这个仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的改动 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

If you want to improve the content directly:
1. Fork this repository
2. Create your feature branch (`git checkout -b feature/AmazingFeature`)
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`)
4. Push to the branch (`git push origin feature/AmazingFeature`)
5. Open a Pull Request

## 内容准则 / Content Guidelines

- 保持内容真实准确 / Keep content authentic and accurate
- 使用清晰的语言表达 / Use clear language
- 遵循现有的格式风格 / Follow existing formatting style
- 保持专业性 / Maintain professionalism

## 代码规范 / Code Standards

如果涉及到代码示例：
- 确保代码可以运行
- 添加必要的注释
- 遵循相应语言的最佳实践

For code examples:
- Ensure code is runnable
- Add necessary comments
- Follow best practices of the respective language
